# Phase 3 — Technical Architecture

## Purpose

This phase defines how the web-first prototype must be built technically.

The correct implementation path is:

```text
Vite + TypeScript + Three.js → browser prototype → mobile browser testing → future PWA/Capacitor packaging
```

Not Unity.

## Files in This Phase

1. `TECHNICAL_SPEC.md`  
   Complete technical architecture and implementation standards.

2. `PROJECT_STRUCTURE.md`  
   Required project folder structure and file responsibilities.

3. `MOBILE_CONTROL_SPEC.md`  
   Touch controls and mobile UI rules.

4. `LOW_END_PERFORMANCE_SPEC.md`  
   Performance constraints for low-end mobile browser and future app packaging.

## Key Principle

The game must be browser-first but mobile-aware from the start.
