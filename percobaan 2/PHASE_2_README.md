# Phase 2 — Game Design

## Purpose

This phase defines the game experience itself: how Petualangan Lumina should feel, what systems exist, how the first quest works, how learning is embedded, and what visual direction the prototype should follow.

## Files in This Phase

1. `GDD.md`  
   Complete game design document for the web-first prototype.

2. `GAME_SYSTEMS_SPEC.md`  
   Detailed breakdown of the systems the AI coding agent must implement.

3. `QUEST_AND_PUZZLE_SPEC.md`  
   Detailed bridge quest and educational puzzle design.

4. `ASSET_DIRECTION.md`  
   Visual, block, character, and UI asset guidance.

## Critical Design Rule

Learning must be part of gameplay. Do not create a separate school quiz screen that breaks immersion.

Correct:

- “Pak Ramu needs 6 wood. Each bridge block uses 2 wood. How many bridge blocks can we craft?” shown with icons while crafting.

Incorrect:

- “Answer this math question to continue” on a detached exam-like page.

## Experience Target

The first prototype should feel like a small magical sandbox where the child can move, explore, gather, help, build, and unlock.
